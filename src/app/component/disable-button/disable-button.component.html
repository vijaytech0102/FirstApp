❓ Why is [disabled] Preferred Over disabled="{{ '{{value}}' }}"?

❌ Not Recommended: disabled="{{ '{{value}}' }}"

<button disabled="{{ '{{isDisabled}}' }}">Click me</button>
  • ⚠️ Produces HTML like: disabled="false" (still disabled!)
  • ❌ Any string in disabled attribute disables the button
  • 🐛 Doesn't work as expected

✅ Recommended: [disabled]="value"

<button [disabled]="isDisabled">Click me</button>
  • ✓ Uses boolean property binding
  • ✓ Correctly disables when value is true
  • ✓ Correctly enables when value is false
  • ⚡ Better performance

Key Takeaway

For boolean attributes like disabled, checked, selected, etc., always use property binding [ ] instead of interpolation.

✅ [disabled]="isDisabled"
✅ [checked]="isChecked"
❌ disabled="{{ '{{isDisabled}}' }}"
❌ checked="{{ '{{isChecked}}' }}"
