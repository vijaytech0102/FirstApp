❓ Difference Between [class.active] and [ngClass]

Aspect [class.className] [ngClass]
Syntax [class.active]="condition" [ngClass]="{{ '{' }} active: condition {{ '}' }}"
Single Class ✅ Simple & Clean ⚠️ More verbose
Multiple Classes ⚠️ Multiple bindings ✅ Single binding
Array Support ❌ No ✅ Yes
Performance ✅ Slightly better for single class Comparable for multiple
Readability ✅ Clear for single class ✅ Clear for multiple classes

Single Class Example

Using [class.active]:

<div [class.active]="isActive">Active Box</div>

Using [ngClass]:

<div [ngClass]="{{ '{' }} active: isActive {{ '}' }}">Active Box</div>

Multiple Classes (Best Practices)

[class.active] - Multiple bindings:

<div [class.active]="isActive"
      [class.highlight]="isHighlighted"
      [class.disabled]="isDisabled">
</div>

[ngClass] - Much cleaner:

<div [ngClass]="{{ '{' }}
  active: isActive,
  highlight: isHighlighted,
  disabled: isDisabled
{{ '}' }}">
</div>
📌 Recommendation:
  • ✅ Use [class.className] for single conditional class
  • ✅ Use [ngClass] for multiple conditional classes
  • ✅ Use [class]="className" for dynamic class names from variables
