Combine Property + Event + Class Binding

🔗 Mixed Concept - Complex Example

Interactive Activate/Deactivate Component

{{ isActive ? 'ACTIVE ✅' : 'INACTIVE ❌' }}

Click the button to toggle the activation state

Current Status {{ isActive ? 'ACTIVE' : 'INACTIVE' }}
Total Toggles {{ clickCount }}

📊 Which Type of Binding is Used in Each Case?

1️⃣ Button Text: "Activate" vs "Deactivate"

Type: Interpolation

Code: {{ '{{ isActive ? "Deactivate" : "Activate" }} Component' }}

Purpose: Display dynamic text based on component state

2️⃣ Div Gets Class "active": [class.active]

Type: Class Binding [ ]

Code: [class.active]="isActive"

Purpose: Conditionally apply CSS class for styling

3️⃣ Button Click Handler: (click)="toggleActive()"

Type: Event Binding ( )

Code: (click)="toggleActive()"

Purpose: Listen to user click and call method

4️⃣ Button Class Changes: [class.btn-deactivate]

Type: Class Binding [ ]

Code: [class.btn-deactivate]="isActive"

Purpose: Change button styling based on state

5️⃣ Display Statistics: {{ clickCount }}

Type: Interpolation

Code: {{ '{{ clickCount }}' }}

Purpose: Display reactive data from component

🎯 Complete Code Structure

Component Class (TypeScript)

export class CombinedBindingComponent {{ '{' }}
  isActive: boolean = false;
  clickCount: number = 0;

  toggleActive(): void {{ '{' }}
    this.isActive = !this.isActive;
    this.clickCount++;
  {{ '}' }}
{{ '}' }}

Template (HTML)

<div [class.active]="isActive">
  <h3>{{ isActive ? 'ACTIVE' : 'INACTIVE' }}</h3>
</div>

<button
  (click)="toggleActive()"
  [class.btn-active]="isActive">
  {{ isActive ? 'Deactivate' : 'Activate' }}
</button>

<p>Clicks: {{ clickCount }}</p>

Styles (CSS)

.active {{ '{' }}
  background-color: #28a745;
  color: white;
{{ '}' }}

.btn-active {{ '{' }}
  background-color: #dc3545;
{{ '}' }}

📋 Summary Table

Binding Type Syntax Purpose Direction
Interpolation {{ '{{ expression }}' }} Display dynamic text/values One-way (Component → Template)
Property Binding [property]="value" Bind to DOM properties One-way (Component → Template)
Event Binding (event)="handler()" Listen to DOM events One-way (Template → Component)
Class Binding [class.name]="condition" Apply classes conditionally One-way (Component → Template)
Two-Way Binding [(ngModel)]="property" Sync between component and form Two-way (Bidirectional)
